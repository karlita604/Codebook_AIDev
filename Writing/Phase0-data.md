# Phase 0 — Data

## Data filtering (`src/phase0/PRfilter.py`)

`PRfilter.py` pulls three tables from `hao-li/AIDev` (pull requests, repositories, users), joins them on `repo_id` / `user_id`, and applies whichever filters are passed in to narrow the full dataset down to a candidate PR list. Filters available:

- `rejected` (default `True`) — closed but not merged
- `star_minimum` (default `100`) — repo stargazer count
- `language` (default `None`) — GitHub Linguist language name(s), comma-separated
- `agents` (default `None`) — coding agent(s), e.g. `Claude_Code`, `Copilot`, `Cursor`, `Devin`, `OpenAI_Codex`
- `user_minimum` (default `None`) — PR author's follower count
- `forks_minimum` (default `None`) — repo fork count
- `age_minimum` / `age_maximum` (default `None`) — days since PR creation

It can be run as a CLI (`python PRfilter.py --star-minimum 500 --language "Python, C, C++"`) or imported as a function (`from PRfilter import filter_prs`). Running it as a script prints the matching PR ids to stdout and writes them to a CSV in `results/phase0/`, named `<MM>-<DD>-<filters>-<count>.csv` (e.g. `07-20-500-pyccpp-1110.csv`, or `07-20-default-7312.csv` when no filter is overridden) — so every filtered pull is reproducible and dated.

### Language filter — valid values

The `language` filter matches GitHub Linguist names exactly (case-sensitive). There are **250 distinct language values** across repositories in the dataset. By repo count, the top ones:

| Language | Repos | Language | Repos |
|---|---|---|---|
| Python | 25,660 | C# | 3,463 |
| TypeScript | 19,709 | Go | 2,511 |
| JavaScript | 15,247 | PHP | 2,483 |
| HTML | 10,336 | C++ | 2,445 |
| Java | 3,481 | Jupyter Notebook | 2,357 |

...continuing down through Rust, CSS, Shell, C, Swift, Kotlin, Dart, Vue, Ruby, Lua, Astro, R, PowerShell, HCL, Blade, SCSS, Svelte, MDX, Dockerfile, TeX, and 220+ longer-tail values (e.g. `1C Enterprise`, `ABAP`, `AGS Script`, `Agda`...).
